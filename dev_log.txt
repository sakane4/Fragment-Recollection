    を追加)。レアドロップ直後ではなく、シーン再生完了後にunlockCompanionする流れに変更(completeAction側の
    即時unlockを撤去、ui.jsのstartCompanionJoinで遅延実行)。
  - 導きパネルのヒントを多段化(該当するものを全部表示): worldLv<5 → 場所発見スケジュール次ステップ →
    forest_gather解放(forestLv2)→ 未発見レアドロップの探索誘導 → 控え中の同行者を誘う → 未装備の固有
    レリックを教える、の優先順つき。
  - 通知ドットシステム: 行動リストの場所見出し(LocationLvが上げられる時)、記憶一覧の各行(更新可能時)、
    フッターの「記憶」タブにバッジ。_seenStories等で既読管理。
  - 行動リストの各行にアイコンを追加(探索/採集/宿屋/花屋/図書館/骨董屋、ラベル文字列で振り分け)。
  - 塔都に「探索」行動(touto_explore)を追加。4施設(宿屋/花屋/図書館/骨董屋)はLocationLv基準ではなく
    touto_exploreのActionLv基準の確率抽選で発見されるように変更(見つからない間は徐々に確率が上昇→1つ
    見つかるとベースに戻る、順序はランダム)。発見ログは「【発見】塔都で「○○」を見つけた」表記に統一。
  - 同行者Lv(ELv)を実際に上げられるように実装。固有フラグメントを消費してレベルアップ(ELV_COSTS、上限
    ELV_MAX=10)。
  - 同行者の異能システムを追加(COMPANION_SKILLS)。ユウヤがLv5で「フラグメント変換」を解放: ノーマル
    フラグメントと任意の固有フラグメント(赤/無色/泡/青/空、所持している物のみ一覧表示)を相互変換できる。
    個数はスライダーで指定、変換は個数×0.2秒かかる時間制タスク(state.companionTasks、ページ再読込にも
    対応)。変換中・同行中はどちらも変換不可(別行動中のみ使用可)。完了時はログに右揃え+リソース色付きで
    結果を表示。
  - 同行者詳細パネルを「にょきっと展開型の縦並び」から、右にアイコンタブ(プロフィール/持ち物/レベル・異能/
    記憶)、左にコンテンツを表示する構成に変更。サブパネルの横フリックでタブ切り替え可能(touch-action:pan-y)。
  - 同行カードの「同行する/別行動」ボタンを廃止し、カード内のドラッグハンドル(⠿)を長押し→ドラッグで
    同行中/別行動エリア間を移動する方式に変更(タップでの開閉トグルとスクロールが競合しないよう、ドラッグ
    操作はハンドル限定に分離)。控え中カードのボタンは「同行する」→「異能」(押すと詳細のレベル・異能タブを
    直接開く)に変更。
  - フッタータブ(記憶/地図/持ち物/同行)を横フリックで切り替えられるように(#sub-panelにスワイプ判定、開発
    タブはスワイプ対象外)。
  - 操作性対応: 全体でテキスト選択を無効化(input/textareaは除く)、overscroll-behavior-y:noneでプルダウン
    更新ジェスチャーを抑制、ヘッダー右端に手動更新ボタンを追加。
  - ブランチ: syusei_own → main、DoukouUI_Kaizen → main、それぞれマージ済み。

--------------------------------------------
■ 実装済み機能
--------------------------------------------

【UIプロトタイプ】
- index.html / game.js / ui.js の3ファイル構成で実装
- ヘッダー、メインパネル(ログ)、アクション行、サブパネル、フッタータブ
- メインパネルとサブパネルの境界をドラッグでリサイズ可能
- カスタムスクロールバー(細め・ゲームカラー)

【セーブ機能】
- localStorage による自動保存・自動復元
- 探索中にページを閉じても、再開時に残り時間から再開

【記憶システム】
- stories.js にパーサー、stories/{id}.js にメタデータ＋本文(body フィールド)を1ファイルで保持
  → 旧 stories/{id}.txt + fetch() 方式は廃止。openStory は story.body を直接パース(同期処理)
  → body は ----- / --- 区切りでページ・段落分割(parseStoryPages)。body が空の記憶は開いてもガードで弾く
- フラグメント消費で物語を解放、段落単位で思い出す
- ポップアップオーバーレイで表示(画面中央)
- showCondition でタブへの出現条件を制御
- ▷/◁ ナビゲーションでページ送り(直接クリック対応)
- 記憶一覧の解放済み行・同行者詳細の関連記憶にも ▷ ボタン(押すとビューアを開く)
- 解放したブロックだけフェードインアニメーション(1.2秒)
  → 同行者Lvアップ通知による二重レンダリングで最終段落のフェードが消える不具合を修正(_viewerFadeUpToの更新を次フレームへ遅延)
- 素材不足時はビューア内トーストで通知
- 進捗表示を「3 / 3」形式に(未開封は「未読」)
- 物語テキストのフォントを小ぶり明朝に変更
- タイトル復元: 最後の段落まで読むと最終ブロックの下に「『(lockedTitle)』を思い出す」ボタンが出る
  → 押すと titleRevealed[id] フラグが立ち、真のタイトルへフェード切替(1秒フェードアウト→1秒フェードイン)
  → 押すまでは lockedTitle(あいまいな記憶)を表示。リスト・ビューア・同行者詳細すべて titleRevealed 基準で判定
- 記憶一覧は titleRevealed 済みを「思い出した記憶」区切りの下に分割表示

【段落コストの動的切り替え】
- 記憶本文(stories/{id}.js の body)内に [pagecost: resource×amount] マーカーを記述
- カンマ区切りで複数素材指定可能: [pagecost: red_fragment×5, forest_voice×2]
- parseStoryCostOverrides() でマーカーを解析 → setStoryCostMap() でキャッシュ
- getCostForParagraph() がランタイムマップを優先参照(フォールバック: pageCost)
- テキストを見ながら境界を直感的に指定可能

【LogSystem】
- 探索中、ランダムな間隔でフレーバーテキストをメインパネルに表示
- 連続重複排除あり
- 探索中にランダムでフラグメントを入手(個数もランダム)
- 同行ボーナス時は +N を緑字で追記
- 同行者がいるとき約55%の確率で同行者ログを表示。固有テキストがある場合はさらに70%でそちらを優先(COMPANION_LOGS_SPECIFIC)

【行動システム】
- 場所(LOCATIONS) → 行動(ACTIONS) の2階層構造
- wherever(ラベルなし): 初期の「探索」の置き場
- forest(はじまりの森) / kyusha(黄昏の旧校舎) / renril(翼竜の都レンリル) / mephisto(魔界王都メフィスト) /
  knights(王立騎士団本部) / touto(塔都): いずれも発見イベントで解放(下記【場所発見システム】参照)
- アクションバーボタン → サブパネルで行動一覧を選択
- 行動完了後、ストーリー発生時以外は自動再開(autoRepeat、下記参照)
- 中断ボタンで即時停止(報酬なし、自動再開なし)
- 行動選択UX: 行をクリックすると選択状態(ハイライト・枠線表示)になり、アクションバーの「開始」ボタンで実行
  (各行の▷開始ボタンは廃止)。選択中に行リスト外をクリックすると、現在実行中の行動の表示に戻る
- 行動セクション(場所ごとの折りたたみ)の開閉状態は、各場所につき初回表示時のみ自動オープンし、以後は
  ユーザーの開閉操作を維持する(ログ更新等の再描画で開閉状態がリセットされないよう修正済み)
- stub: true の行動(塔都の4施設)は完了時「まだ準備中だ・・・」を表示するだけで報酬なし

【行動の一時停止・再開】
- pauseAction() / resumeAction() でタイマーを保存・復元
- ログストーリー開始時に自動ポーズ、終了時に自動再開
- getProgress() が pausedAt を参照して進捗を静止
- tick() がログ再生中はプログレスバー更新をスキップ

【場所発見システム】
- 再生された世界(wherever)のLocationLvが5/10/15/20/25に達するたびに、メインパネルにログ風の選択イベント
  (runLocationChoice、typewriter＋選択ボタン)が発生し、新しい場所(と対応する行動)を選択・解放できる
  - Lv5: 黄昏の旧校舎・翼竜の都レンリルから1つを選択(序盤2択)
  - Lv10: 塔都(固定で解放、選択肢なし)。このタイミングで終盤2locationの抽選(1/2)も裏で確定する
  - Lv15: 序盤の残り1つ＋終盤の抽選で決まった1つから選択
  - Lv20: 残り2つから選択
  - Lv25: 残り1つを自動解放(最後の1つなので選択肢なし)
- getPendingDiscovery(state) / resolveDiscovery(chosenId) (game.js) がスケジュールの判定・解決を担当
- discover_location_choiceルール(rules.js): repeatable: true で各ステップごとに再発火する
- _choicePending(ui.js)で同フレーム内の多重発火を防止

【同行システム】
- unlockedCompanions / activeCompanions で状態管理
- 同行タブに「同行中」「控え(別行動)」の2エリア
- 同行者がいると探索報酬が ×(1+同行人数) になる(以前は同行者がいれば常に×2固定だったが、人数に応じて
  スケールするよう変更)
- 新同行者解放時にタブ上にトーストポップアップ表示
- 同行者固有報酬(完了時・ランダム)
- discoveredResources: 一度でも入手したリソースを記録
- 同行者詳細「にょきっと展開型」: 一覧の行をクリックすると、その行の下にアコーディオンで詳細を表示(複数人同時に展開可)
  → プロフィール / 固有ドロップ品(未発見時は ??? 表示・discoveredResources基準) / 持ち物 / 関連する記憶(companionIdで絞り込み・▷で開く)
- 持ち物(装備品、本実装済み): ボタンを押すと装備候補ポップアップ。「外す」+所持している場合のみその同行者
  固有のレリック(COMPANION_RELICS)を選択可。装備中は固定報酬に+EQUIP_BONUS(5)。state.companionEquipment で管理
- ユウヤ以外の4人(ラビ・シズク・カオル・雪架)は、対応する場所の探索でレアドロップ(5%)を入手した後、
  加入イベント専用シーン(scenario.jsのJOIN_*_STEPS、typewriter)が再生され、完了時にunlockedCompanionsに
  追加される(ui.jsのstartCompanionJoinが司る。completeAction側での即時unlockは行わない)
  - 黄昏の旧校舎 → 旧美術室の鍵 → シズク
  - 翼竜の都レンリル → メロンパンのキーホルダー → カオル
  - 魔界王都メフィスト → 天空の羅針盤 → 雪架
  - 王立騎士団本部 → 美しい細身の剣 → ラビ
- 同行カードのUI: 「同行する/別行動」ボタンは廃止。カード内のドラッグハンドル(⠿)を長押し→ドラッグで
  同行中/別行動エリア間を移動(_attachPartyDragHandlers、ui.js)。短いタップ(ドラッグなし)はカード本体への
  クリックで詳細パネルの開閉トグル。控え中カードには「異能」ボタン(詳細のレベル・異能タブを直接開く)
- 同行者詳細パネル: 右にアイコンタブ(プロフィール/持ち物/レベル・異能/記憶)、左にコンテンツを表示する構成
  (_buildCompanionDetail)。コンテンツ領域は横フリックでタブ切り替え可能(_attachSwipeTabHandlers、
  touch-action:pan-y)

【同行者Lvシステム】
- 記憶のページ(-----区切り)を全段落解放するたびに対応キャラのLvが+1
- stories/{id}.js の companionId フィールドで紐づけ
- ELv: { companionId: number } をstateで管理・セーブ対応
- 固定報酬に Lv 分を加算(例: Lv3 → +3 ボーナス)
- 同行者メニューの名前横に「Lv N」を表示(Lv0は非表示)
- ログに「【同行者】〇〇の存在が少し安定した (Lv N)」を出力
- レベルアップ操作(本実装済み): 同行者詳細「レベル」タブの「レベルアップ」ボタンを押すと、固有フラグメント
  をELV_COSTS[lv]分消費してELvが+1(levelUpCompanion、game.js)。上限ELV_MAX=10。物語の自動Lvアップとは
  別経路で同じELvを操作する

【同行者の異能システム】
- COMPANION_SKILLS(game.js)で同行者ごとの異能を定義。ELvが指定Lvに達すると解放される
- ユウヤ: Lv5で「フラグメント変換」を解放。ノーマルフラグメントと任意の固有フラグメント(赤/無色/泡/青/空、
  所持しているものだけ一覧表示)を相互に1:1変換できる
- 変換ポップアップ(#convert-popup): 上のボックスが変換元、下が変換先。ボックスをクリックすると選択状態
  になり、その状態でリストから選んだリソースがそのボックスに入る(上下入れ替えクリックは廃止)。個数は
  スライダーで指定(自由入力ではない)。未所持の固有フラグメントはリストに出さない
- 変換は個数×FRAGMENT_CONVERT_MS_PER_UNIT(200ms)の時間制タスク。state.companionTasks[companionId]で管理し、
  ページ再読込後も残り時間から再開する(init()で再スケジュール)
- 制約: 変換中はその同行者を同行させられない／同行中はその同行者の異能を使えない(setActiveCompanionと
  startFragmentConvertの両方で相互にガード)
- 完了時、stateのlastCompanionTaskResultをui.js側でdiff検知し、ログに右揃え(.log-entry--right)＋
  resourceSpanで色付けした結果メッセージを1回だけ出力

【リソース管理】
- RESOURCE_LABELS / RESOURCE_COLORS を ui.js で一元管理
- resourceSpan(resource, text) でアイテム名に色を自動付与
- 新アイテム追加時は RESOURCE_LABELS・RESOURCE_COLORS・INITIAL_STATE.resources に1行追加するだけ
- 木々の声(forest_voice): 探索・採集共通でランダム入手(forest_common_random) + 探索固有でさらに追加入手、Lv2以降は探索分の最大量が増加(forest_explore_random)
- 薬草(herb)・フラグメント・木の枝(branch、Lv2以降): はじまりの森の採集でランダム入手(forest_gather_random)
- 各キャラ場所のドロップ品: 旧校舎(古びた絵具/破れたページ/少し狂ったピアノの音)、レンリル(翼竜の爪/翼竜の鱗)、
  メフィスト(魔術書のページ/魔法陣の欠片/アスタード文字の破片)、騎士団本部(討伐報告書/古びた腕章/欠けた記章)

【チュートリアル】
- オープニング: 4ページのテキスト演出(タイプライター)
  - tap / button の2種類の進め方
  - 右上コーナータップで隠しスキップ
  - ボタンはテキスト完了1秒後に表示
- アクションガイド: チュートリアル完了後にアクションバー上にポップアップ
- フロー: 初回探索完了 → プロローグ強制解放 → 全ページ読む → 001 → ユウヤ加入

【通知・UX】
- タブトースト: 指定タブ上に吹き出しポップアップ(3秒で消える)
- プログレスバー: アクションバーの上に配置
- ログ自動スクロール: 下端付近のときのみ
- worldLv進捗ポップアップ: ヘッダーのworldLv表示をクリックすると、次のworldLvまでの進捗バーが表示される

【条件管理(rules.js)】
- UNLOCK_RULES 配列でアンロック条件を一元管理
- evaluateRules(state, ctx) を render() 末尾で毎フレーム評価
- requireViewerClosed: true でビューアを閉じたタイミングに限定
- repeatable: true のルールは _fired に登録せず、条件を満たすたびに再発火できる(discover_location_choiceで使用)
- イベント駆動条件(探索完了後など)は ui.js コールバックに残す(ハイブリッド方式)

【worldLvシステム】
- フラグメントの総獲得数(totalFragments)でworldLvが上がる
- 閾値: 25段階、WORLD_LV_THRESHOLDS = 50*(n²+n+1) の計算式で自動生成(旧: [50,150,350,700,1200]の5段階)
- worldLvが上がるとフラグメント獲得量が増加(REWARD_TABLES関数内で参照)
- ヘッダー右端にworldLv表示。クリックで進捗ポップアップを表示

【LocationLvシステム】
- 場所ごとのレベル(Lv0〜25)、state.LocationLv: { locationId: number }
- コスト: LOCATION_LV_COSTS = 50*(n²+n+1) の計算式で自動生成(25段階)
- LocationLvの上限はworldLvが天井になる(getLocationLvCap() = min(25, worldLv))。worldLvが足りないと
  「今はこれ以上再生できない...」とボタン内に表示され、レベルアップ不可
- 場所詳細ポップアップ(infoアイコン)でレベルアップ操作
- 長押しでプログレスバーが伸び、フラグメントをリアルタイム消費
- 途中で離すと一時停止(進捗・消費は保持)、場所ごとに独立した永続進捗
- 100%到達→ボタンが白文字青背景に変化→タップで確定
- レベルアップ確定時は、ログへの追記(LocationLvアップ)→notify()の順で処理し、同フレームで発見イベントが
  発火しても「LocationLvが上がった」ログが先に出るよう順序を保証(levelUpLocationにsilentオプションを追加)
- Lv2以降など閾値で報酬テーブルが変化(REWARD_TABLES関数内のlocationLvで制御)
- レベルアップボタンは所持数に関わらず常に押せる(disabled誤判定を修正)、ボタン上部に「所持数：〇〇」を表示(長押し中リアルタイム更新)
- 場所名と現在のLvをポップアップ上部に並べて表示(location-popup-header)

【ActionLvシステム】
- 行動ごとのレベル。実行回数の累計でレベルアップ。state.actionCount / state.ActionLv で管理
- 閾値: ACTION_LV_THRESHOLDS = [10, 30, 60, 100, 150]（仮値、LocationLvとは別管理で25段階化はしていない）
- completeAction内で_addActionCount(actionId)を呼ぶ。LocationLvと同様、報酬テーブルに actionLv を渡して反映
- 行動一覧の各行名の横に「LvN」を常時表示(Lv0含む)

【REWARD_TABLES関数化】
- テーブルを (state, locationLv, actionLv) => rewards の関数形式に変更
- worldLv / locationLv / actionLv を報酬計算に反映可能(resolveTable が locationId と actionId を解決)
- 各アクションは rewardTable / rewardTableRandom でテーブル名を参照
- 直接報酬(rewards/randomRewards)との併用も可能

【行動メニューリデザイン】
- 場所ごとにセクション(折りたたみ)、行動を行として並べる
- 場所名横のinfoアイコンで場所詳細ポップアップを表示
- ポップアップは画面端をはみ出さないよう自動位置補正

【導き(ヒント)パネル・オート機能】
- メインパネル右側に「導き」タブが少しだけ突出して表示、押すと横からパネルがスライドして開く
- パネル外をクリックすると自動で閉じる
- ログストーリー004読了(logSt4Done)で解放(unlock_guideルール)。解放前はタブ自体を非表示(class="hidden")
- 解放時にログへ「【導き】が解放された」「これからは行動が自動でくり返されるようになった」を出力し、
  同時に state.autoRepeat を自動でtrueにする(一度解放したらオートを使わない選択はほぼ無いという判断で、
  専用のオートトグルボタンは廃止し「開始」ボタンを押すこと自体がオート運用になる形にした)
- パネル内容: renderGuideList(state)でヒントを描画。該当する条件をすべて表示する多段ヒント方式(優先順):
  1. worldLv<5 → 「再生された世界」を再生しよう(Lv5)
  2. 場所発見スケジュールの次ステップ(DISCOVERY_STEP_LV)に向けて再生を促す
  3. forest(はじまりの森)解放済み・forest_gather未解放・forestLv<2 → 採集解放の示唆
  4. 解放済みの場所に未発見のレアドロップ(同行者)があれば探索を促す
  5. 塔都解放済み・未発見の施設があれば塔都の探索を促す
  6. 控え中(別行動)の同行者がいれば同行を誘う
  7. 同行中なのに固有レリックを未装備の同行者がいれば装備を教える
  8. 該当なしの場合のみ「星の導きは、いまは静かだ…」
- 実装時の注意: パネルに overflow-y:auto を直接付けると、はみ出して見せたいタブボタン(left:-28pxで配置)が
  CSS仕様上 overflow-x も自動でクリップされてしまうため、内側に #guide-panel-inner を新設してスクロールを分離した

【通知ドットシステム】
- _seenStories(Set、localStorage永続)等で「既読」を管理。記憶タブを開くと既読化(_markStoriesSeen)
- 行動リストの場所見出しに、その場所のLocationLvが上げられる時の通知ドット(_canLevelUpLocation)
- 記憶一覧の各行に、次ページ解放/新規解放が可能な時の通知ドット(_storyIsUpdatable)
- フッターの「記憶」タブに、未読の新着記憶がある時のバッジ(_updateStoriesBadge)

【塔都・施設発見システム】
- 塔都にも「探索」行動(touto_explore)があり、フラグメント獲得に加えて4施設(宿屋尻尾亭/花屋竜の鱗/
  塔都図書館/骨董屋リーリエ、いずれもstub:trueで報酬は未実装)をランダムな順で発見していく
- 発見順序(toutoFacilityOrder)は初回探索時にシャッフルして確定・セーブに保持
- 発見はLocationLv基準ではなく、touto_exploreのActionLv基準の確率抽選(completeAction内)。未発見の間は
  ActionLvとの差分に応じて確率が上昇(base5%+差分×10%、上限50%)、1つ見つかるとtoutoLastFacilityLvが
  更新されて確率がベースに戻る
- 発見ログは他の場所と違い「【発見】塔都で「○○」を見つけた」表記(action.stub判定で文言を分岐)

【操作性・PWA寄りの調整】
- html/body/#app に user-select:none, -webkit-touch-callout:none(input/textareaは除外)でテキスト選択を
  全体的に無効化(長押し操作と競合するため)
- html/bodyに overscroll-behavior-y:none でブラウザのプルダウン更新ジェスチャーを抑制
- ヘッダー右端に手動更新ボタン(#refresh-btn、押すとlocation.reload())を常設
- 同行カードのドラッグハンドル(.party-drag-handle)のみ touch-action:none、カード本体は touch-action:auto
  にしてスクロールとドラッグ判定の競合を回避
- #sub-panel(記憶/地図/持ち物/同行)・同行者詳細のコンテンツ領域は touch-action:pan-y で横フリック切り替え
  に対応(縦スクロールは普通に効く)

■ 最適化案
--------------------------------------------
- **`render()` が重い** — state 変化のたびに全タブを再描画しているが、今の規模では問題なし
- **ui.js / index.html が大きくなってきた**(それぞれ2000行超)。1ターンの編集コストが上がってきているので、
  同行タブ関連(companion-detail, drag&drop, convert popup等)を別ファイルに切り出すのも検討候補
- 長く続いたセッションは会話履歴自体も重くなるので、機能が一段落するたびに新しいチャットに切り替えるのが
  トークン節約として手軽(このdev_log.txtを最初に読んでもらえば経緯を引き継げる)
- **logs.jsの[lv:n]タグパース** — ACTION_LOGSの本文行から正規表現でminLvを毎回パースしている(_parseLvTag)。
  今の規模(数十行・呼び出し頻度も低い)では実害なしだが、リリース前に気になれば対応candidate:
  (a) actionIdごとにパース結果をメモ化(Map)してパースを1回だけにする、
  (b) ビルド時に[lv:n]タグを{ text, minLv }の静的配列へ事前変換してランタイムパースを無くす
